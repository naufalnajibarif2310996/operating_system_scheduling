Pengantar Penjadwalan Proses

Penjadwalan proses adalah aktivitas inti dalam manajemen sistem operasi yang menentukan proses mana yang akan mendapatkan CPU dan kapan. Penjadwalan proses merupakan bagian fundamental dari sistem operasi multiprogramming yang memungkinkan beberapa proses berjalan secara bersamaan dengan berbagi sumber daya CPU.

Dalam sistem komputer modern, CPU adalah sumber daya yang sangat berharga. Penjadwalan proses memastikan bahwa CPU digunakan secara efisien dan semua proses mendapatkan kesempatan yang adil untuk dieksekusi.

Tujuan utama penjadwalan proses adalah untuk:

  • Memaksimalkan throughput (jumlah proses yang diselesaikan per unit waktu)
  • Meminimalkan waktu tunggu (waktu yang dihabiskan proses dalam antrian)
  • Meminimalkan waktu respons (waktu dari permintaan hingga respons pertama)
  • Memaksimalkan pemanfaatan CPU (menjaga CPU tetap sibuk)
  • Memberikan keadilan bagi semua proses

Konsep Dasar

Siklus Hidup Proses

Setiap proses dalam sistem operasi memiliki siklus hidup yang terdiri dari beberapa status:

  • New: Proses baru dibuat
  • Ready: Proses siap untuk dieksekusi dan menunggu CPU
  • Running: Proses sedang dieksekusi oleh CPU
  • Waiting/Blocked: Proses menunggu suatu peristiwa (seperti I/O)
  • Terminated: Proses selesai dieksekusi

CPU Burst dan I/O Burst

Eksekusi proses terdiri dari siklus CPU burst (waktu eksekusi di CPU) dan I/O burst (waktu menunggu operasi I/O):

  • CPU Burst: Periode saat proses menggunakan CPU untuk komputasi
  • I/O Burst: Periode saat proses menunggu operasi I/O selesai

Proses dapat dikategorikan berdasarkan karakteristik burst-nya:

  • CPU-bound: Proses dengan CPU burst yang panjang dan I/O burst yang pendek
  • I/O-bound: Proses dengan CPU burst yang pendek dan I/O burst yang panjang

Dispatcher

Dispatcher adalah modul yang memberikan kontrol CPU kepada proses yang dipilih oleh penjadwal CPU. Fungsinya meliputi:

  • Context switching (menyimpan dan memulihkan konteks proses)
  • Beralih ke mode pengguna
  • Melompat ke lokasi yang tepat dalam program pengguna untuk melanjutkan eksekusi

Dispatch latency adalah waktu yang dibutuhkan dispatcher untuk menghentikan satu proses dan memulai proses lain.

Kriteria Penjadwalan

Algoritma penjadwalan CPU dievaluasi berdasarkan beberapa kriteria kinerja:

CPU Utilization

Persentase waktu CPU sibuk. Idealnya mendekati 100%.

Throughput

Jumlah proses yang diselesaikan per unit waktu.

Turnaround Time

Waktu total dari saat proses masuk sistem hingga selesai.

Turnaround Time = Completion Time - Arrival Time

Waiting Time

Total waktu proses menunggu dalam antrian ready.

Waiting Time = Turnaround Time - Burst Time

Response Time

Waktu dari pengajuan proses hingga respons pertama.

Response Time = First CPU Time - Arrival Time

Algoritma penjadwalan yang baik harus meminimalkan waiting time, turnaround time, dan response time, serta memaksimalkan CPU utilization dan throughput.

Jenis Penjadwalan

Berdasarkan Waktu Pengambilan Keputusan

Jenis Deskripsi Karakteristik
Preemptive Proses yang sedang berjalan dapat diinterupsi dan dipindahkan ke status ready
  • Responsif terhadap proses baru
  • Cocok untuk sistem time-sharing
  • Overhead lebih tinggi karena context switching lebih sering
Non-preemptive Proses yang sedang berjalan tidak dapat diinterupsi hingga selesai atau melakukan I/O
  • Lebih sederhana dan overhead lebih rendah
  • Waktu respons kurang dapat diprediksi
  • Cocok untuk sistem batch

Berdasarkan Lingkungan Sistem

Jenis Deskripsi Contoh Algoritma

Interactive Scheduling Mengoptimalkan response time dan memberikan keadilan bagi pengguna interaktif Round Robin, Priority Scheduling
Real-time Scheduling Memenuhi deadline dan persyaratan waktu yang ketat Rate Monotonic, Earliest Deadline First

Algoritma Penjadwalan

Berikut adalah algoritma penjadwalan CPU yang umum digunakan:

First-Come, First-Served (FCFS)

FCFS adalah algoritma penjadwalan paling sederhana yang mengeksekusi proses sesuai urutan kedatangan mereka.

Karakteristik:

  • Non-preemptive
  • Implementasi sederhana dengan antrian FIFO
  • Dapat menyebabkan convoy effect (proses pendek menunggu proses panjang)
  • Tidak optimal untuk sistem interaktif

Contoh:

Proses Arrival Time Burst Time
P1 0 24
P2 1 3
P3 2 3
P1 (0-24)
P2 (24-27)
P3 (27-30)

Waiting Time: P1 = 0, P2 = 23, P3 = 25
Average Waiting Time = (0 + 23 + 25) / 3 = 16

Shortest Job First (SJF)

SJF mengeksekusi proses dengan waktu burst terpendek terlebih dahulu. Ini optimal untuk meminimalkan waktu tunggu rata-rata.

Karakteristik:

  • Dapat preemptive (SRTF) atau non-preemptive
  • Optimal untuk meminimalkan average waiting time
  • Sulit diimplementasikan karena membutuhkan prediksi burst time
  • Dapat menyebabkan starvation untuk proses dengan burst time panjang

Contoh (Non-preemptive):

Proses Arrival Time Burst Time
P1 0 6
P2 1 3
P3 2 8
P4 3 4
P1 (0-6)
P2 (6-9)
P4 (9-13)
P3 (13-21)

Waiting Time: P1 = 0, P2 = 5, P3 = 11, P4 = 6
Average Waiting Time = (0 + 5 + 11 + 6) / 4 = 5.5

Shortest Remaining Time First (SRTF)

SRTF adalah versi preemptive dari SJF. CPU selalu dialokasikan ke proses dengan sisa waktu eksekusi terpendek.

Karakteristik:

  • Preemptive
  • Optimal untuk meminimalkan average waiting time
  • Overhead context switching yang tinggi
  • Sulit diimplementasikan karena membutuhkan prediksi sisa waktu eksekusi

Contoh:

Proses Arrival Time Burst Time
P1 0 8
P2 1 4
P3 2 9
P4 3 5
P1 (0-1)
P2 (1-5)
P4 (5-10)
P1 (10-17)
P3 (17-26)

Waiting Time: P1 = 9, P2 = 0, P3 = 15, P4 = 2
Average Waiting Time = (9 + 0 + 15 + 2) / 4 = 6.5

Round Robin (RR)

Round Robin adalah algoritma penjadwalan yang memberikan jatah waktu CPU yang sama (time quantum) untuk setiap proses secara bergiliran.

Karakteristik:

  • Preemptive
  • Adil untuk semua proses
  • Responsif untuk sistem interaktif
  • Kinerja sangat bergantung pada ukuran time quantum
  • Overhead context switching yang tinggi

Contoh (Time Quantum = 4):

Proses Arrival Time Burst Time
P1 0 10
P2 0 5
P3 0 8
P1 (0-4)
P2 (4-8)
P3 (8-12)
P1 (12-16)
P2 (16-17)
P3 (17-21)
P1 (21-23)

Waiting Time: P1 = 13, P2 = 12, P3 = 13
Average Waiting Time = (13 + 12 + 13) / 3 = 12.67

Priority Scheduling

Priority Scheduling mengalokasikan CPU ke proses berdasarkan prioritas yang ditetapkan.

Karakteristik:

  • Dapat preemptive atau non-preemptive
  • Mendukung prioritas proses yang berbeda
  • Dapat menyebabkan starvation untuk proses dengan prioritas rendah
  • Membutuhkan mekanisme aging untuk mencegah starvation

Contoh (Non-preemptive, prioritas rendah = angka kecil):

Proses Arrival Time Burst Time Prioritas
P1 0 7 3
P2 2 4 1
P3 4 1 4
P4 5 4 2
P1 (0-2)
P2 (2-6)
P4 (6-10)
P1 (10-15)
P3 (15-16)

Waiting Time: P1 = 3, P2 = 0, P3 = 11, P4 = 1
Average Waiting Time = (3 + 0 + 11 + 1) / 4 = 3.75

Multilevel Queue

Multilevel Queue membagi proses ke dalam beberapa antrian berdasarkan properti tertentu, dengan algoritma penjadwalan yang berbeda untuk setiap antrian.

Karakteristik:

  • Proses ditetapkan ke antrian tertentu secara permanen
  • Setiap antrian memiliki algoritma penjadwalan sendiri
  • Penjadwalan antar antrian biasanya menggunakan prioritas tetap
  • Dapat menyebabkan starvation untuk antrian prioritas rendah

Contoh Struktur Multilevel Queue:

Foreground Queue (Interaktif)
Round Robin
Prioritas Tinggi
Background Queue (Batch)
FCFS
Prioritas Rendah

Dalam contoh ini, proses interaktif dijalankan dengan Round Robin di antrian foreground dengan prioritas tinggi, sementara proses batch dijalankan dengan FCFS di antrian background dengan prioritas rendah.

Kelebihan dan Kekurangan

FCFS

Kelebihan:
  • Sederhana dan mudah diimplementasikan
  • Tidak ada starvation
  • Adil berdasarkan urutan kedatangan
Kekurangan:
  • Convoy effect
  • Waktu tunggu rata-rata tinggi
  • Tidak cocok untuk sistem interaktif

SJF/SRTF

Kelebihan:
  • Optimal untuk waktu tunggu rata-rata
  • Meningkatkan throughput
  • Mengurangi waktu tunggu untuk proses pendek
Kekurangan:
  • Sulit memprediksi burst time
  • Dapat menyebabkan starvation untuk proses panjang
  • SRTF memiliki overhead context switching yang tinggi

Round Robin

Kelebihan:
  • Adil untuk semua proses
  • Response time baik
  • Tidak ada starvation
  • Cocok untuk sistem time-sharing
Kekurangan:
  • Performa tergantung pada quantum time
  • Overhead context switching tinggi
  • Waktu tunggu rata-rata lebih tinggi dari SJF

Priority Scheduling

Kelebihan:
  • Fleksibel untuk berbagai kebutuhan
  • Dapat mengakomodasi urgensi proses
  • Cocok untuk sistem real-time
Kekurangan:
  • Dapat menyebabkan starvation
  • Memerlukan mekanisme aging
  • Overhead penentuan prioritas

Studi Kasus

Kasus 1: Sistem Batch

Dalam sistem batch seperti server pemrosesan data, tujuan utama adalah memaksimalkan throughput dan meminimalkan turnaround time. Algoritma SJF sering digunakan karena optimal untuk meminimalkan waktu tunggu rata-rata.

Contoh: Server Pemrosesan Data

Sebuah server pemrosesan data menerima berbagai pekerjaan dengan waktu eksekusi yang berbeda-beda. Dengan menggunakan SJF, pekerjaan-pekerjaan kecil dapat diselesaikan lebih cepat, meningkatkan throughput keseluruhan.

Hasil: Peningkatan throughput sebesar 30% dibandingkan dengan FCFS.

Kasus 2: Sistem Interaktif

Dalam sistem interaktif seperti desktop OS, tujuan utama adalah memberikan response time yang baik untuk pengguna. Round Robin sering digunakan karena memberikan keadilan dan response time yang baik.

Contoh: Sistem Operasi Desktop

Sistem operasi desktop menjalankan berbagai aplikasi interaktif seperti browser, editor teks, dan media player. Dengan menggunakan Round Robin, semua aplikasi mendapatkan jatah CPU secara bergiliran, memberikan pengalaman yang responsif bagi pengguna.

Hasil: Response time yang konsisten dan pengalaman pengguna yang lebih baik.

Kasus 3: Sistem Real-time

Dalam sistem real-time seperti kontrol industri, tujuan utama adalah memenuhi deadline. Priority Scheduling sering digunakan dengan prioritas berdasarkan deadline.

Contoh: Sistem Kontrol Industri

Sistem kontrol industri memiliki berbagai tugas dengan tingkat kekritisan yang berbeda. Dengan menggunakan Priority Scheduling, tugas-tugas kritis dapat dieksekusi terlebih dahulu, memastikan deadline terpenuhi.

Hasil: 99.9% deadline terpenuhi, meningkatkan keandalan sistem.
